## 三、集成测试方案

### 3.1 API 集成

1. 启动 PostgreSQL 和 Redis。
2. 启动 API。
3. 初始化管理员。
4. 创建项目空间、应用、部署配置。
5. 创建 Git Provider、Git Account mock 或测试凭据。
6. 触发构建并断言 BuildRun/BuildJob 状态。
7. 创建 Release 并断言状态流转。
8. 查询账单和日志接口。

### 3.2 Worker 集成

1. 使用本地 kind/minikube/K3s 作为运行集群。
2. 创建测试 namespace。
3. 用一个最小 Dockerfile 触发构建。
4. 验证 BuildKit Job 创建、日志写入、镜像推送。
5. 验证自动部署和 Service/Ingress 生成。

## 五、边界条件测试

- 项目 slug 和应用 slug 冲突。
- 部署配置名称包含中文或超长文本。
- Dockerfile 路径不存在。
- 镜像 tag 重复使用 `latest`。
- Kubernetes API 不可达。
- DNS、镜像站或包管理网络不可达。
- Redis 不可用。
- 用户余额不足。
- Secret 留空更新。
- OIDC 回调 state 过期。
