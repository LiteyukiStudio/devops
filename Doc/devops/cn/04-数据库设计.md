## 一、数据库概述

系统使用 PostgreSQL 作为主数据库，GORM 作为 ORM，golang-migrate 负责 SQL 迁移。API 和 worker 共用同一 schema。Redis 仅用于异步任务、缓存和限流，不作为业务状态最终来源。

迁移文件位于 `migrations/`，当前包含 baseline、构建资源、部署目标、计费 owner、应用模板安装记录等迁移。

## 二、核心数据表

### 2.1 用户与认证

| 模型 | 说明 | 关键字段 |
| --- | --- | --- |
| User | 平台用户 | id、username、email、role、language、avatar |
| UserSession | 登录会话 | userId、tokenHash、expiresAt |
| UserRememberToken | 恢复登录令牌 | userId、tokenHash、expiresAt |
| AccessToken | API Token | userId、tokenHash、scopes、expiresAt、revokedAt |
| AuthProvider | OIDC Provider | type、issuer、clientId、clientSecretRef |
| ExternalIdentity | 第三方身份绑定 | userId、providerId、subject、email |
| AuthAdmissionPolicy | 登录准入策略 | allowedDomains、allowedEmails、requiredGroups |

### 2.2 项目空间与应用

| 模型 | 说明 | 关键字段 |
| --- | --- | --- |
| Project | 项目空间 | slug、name、namespaceStrategy、billingOwnerUserId、deleteStatus |
| ProjectMember | 项目成员 | projectId、userId、role、lastUsedAt、useCount |
| ProjectPin | 用户固定项目 | userId、projectId、pinnedAt |
| Application | 应用 | projectId、slug、name、icon、deleteStatus |
| AppConfig | 站点配置 | key、value |
| ProjectHookConfig | 项目 Hook 配置 | projectId、script、phase、failurePolicy |
| HookRun | Hook 运行记录 | buildRunId、releaseId、status、exitCode |
| HookRunLog | Hook 日志 | hookRunId、content |

### 2.3 构建与部署

| 模型 | 说明 | 关键字段 |
| --- | --- | --- |
| RuntimeCluster | 运行集群 | name、type、scope、kubeconfigRef、isDefault、status |
| Environment | 环境 | projectId、slug、clusterId、namespace |
| DeploymentTarget | 部署配置 | applicationId、clusterId、sourceType、imageRef、runtime config |

### 2.5 网关与计费

## 三、表间关系

```text
User 1 -> N ProjectMember N -> 1 Project
Project 1 -> N Application
Application 1 -> N DeploymentTarget
DeploymentTarget 1 -> N BuildRun
BuildRun 1 -> N BuildJob
BuildRun 1 -> N Release
DeploymentTarget 1 -> N Release
Project 1 -> N GatewayRoute
Project 1 -> N BillingUsageRecord
User 1 -> 1 UserWallet
User 1 -> N BillingLedgerEntry
```

## 四、关键唯一性与索引

| 模型 | 约束 |
| --- | --- |
| Project | `slug` 在未删除记录中全局唯一 |
| Application | `projectId + slug` 在未删除记录中唯一 |
| ProjectMember | 建议保证 `projectId + userId` 唯一 |
| ProjectPin | `userId + projectId` 唯一 |
| BillingRateRule | `meter` 唯一 |
| BillingUsageRecord | `resourceType + resourceId + meter` 唯一 |
| RegistryCredential | 按 registry 维度管理，Secret 字段不回显 |
| BuildJob | 通过 `buildRunId` 和状态索引支持查询 |

## 五、删除策略

- Project、Application、DeploymentTarget、RuntimeCluster 等资源使用 GORM soft delete。
- deleteStatus 用于异步删除流程展示，包括 active、deleting、failed 等状态。
- 应用或项目删除不删除账单、用量和账本。
- Secret 引用由资源删除任务清理，但历史日志不回显明文。

## 六、迁移策略

- API 和 worker 启动时必须确保迁移执行到最新版本。
- 迁移文件只向前追加，不修改已发布迁移。
- 结构变更涉及数据回填时使用独立 migration。
- 大字段 JSON 字符串当前保存在 text 字段，后续可按查询需求迁移为 jsonb。
