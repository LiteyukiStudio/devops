# 第一部分：部署手册

## 一、环境要求

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Linux/macOS/Windows Docker 环境 |
| 容器运行时 | Docker + Docker Compose |
| 数据库 | PostgreSQL 17 或兼容版本 |
| 缓存/队列 | Redis 8 或兼容版本 |
| 运行集群 | Kubernetes/K3s，需提供 kubeconfig |
| 证书 | 如需 HTTPS，建议安装 cert-manager |
| 镜像站 | DockerHub、Harbor、Gitea Registry 或兼容 OCI Registry |

## 二、快速部署

使用 Docker Compose 启动完整平台：

```bash
docker compose up -d
```

默认访问地址：

```text
http://localhost:8088
```

默认镜像：

```text
liteyukistudio/devops-api:${DEVOPS_IMAGE_TAG:-nightly}
liteyukistudio/devops-worker:${DEVOPS_IMAGE_TAG:-nightly}
```

指定版本：

```bash
DEVOPS_IMAGE_TAG=v0.1.0 docker compose up -d
```

## 三、关键配置项

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| `APP_ENV` | `development` | 运行模式，生产建议 `production` |
| `API_ADDR` | `:8080` | API 监听地址 |
| `DATABASE_URL` | compose 内置 PostgreSQL | 数据库连接串 |
| `REDIS_ADDR` | `redis:6379` | Redis 地址 |
| `PUBLIC_BASE_URL` | `http://localhost:8088` | 外部访问地址，用于回调和链接生成 |
| `APP_CORS_ORIGINS` | `http://localhost:8088` | 允许跨域来源 |
| `SECRET_ENCRYPTION_KEY` | 空 | 生产必须设置稳定随机值 |
| `BUILD_EXECUTOR_IMAGE` | `moby/buildkit:v0.24.0-rootless` | 构建执行器镜像 |
| `BUILD_EGRESS_MODE` | `permissive` | 构建出站网络模式 |
| `BUILD_PRIVATE_EGRESS_CIDRS` | 空 | 构建允许访问的私有网段 |
| `BUILD_PRIVATE_EGRESS_PORTS` | `443` | 私有网段允许端口 |
| `CERT_MANAGER_CLUSTER_ISSUER` | `letsencrypt-http01` | 证书 issuer |

## 四、本地开发

启动开发依赖：

```bash
docker compose -f docker-compose-dev.yaml up -d --build
```

启动 API：

```bash
go run ./cmd/api
```

启动 worker：

```bash
go run ./cmd/worker
```

启动前端：

```bash
pnpm --dir web install
pnpm --dir web dev
```

启动文档站：

```bash
pnpm --dir docs install
pnpm --dir docs dev --port 5274
```

## 五、生产初始化

1. 启动平台。
2. 打开 `PUBLIC_BASE_URL`。
3. 首次进入时创建管理员。
4. 在系统设置中配置站点 title、logo、favicon。
5. 配置 OIDC Provider 和准入策略。
6. 配置运行集群。
7. 配置镜像站和凭据。
8. 创建项目空间和应用。

# 第二部分：用户手册

## 一、部署一个 Web 项目

1. 创建项目空间。
2. 创建应用。
3. 连接 GitHub 或 Gitea 账号。
4. 在应用中绑定仓库。
5. 创建部署配置，填写 Dockerfile、构建上下文、镜像站、端口和资源。
6. 点击触发构建。
7. 构建成功后创建 Release，或开启自动部署。
8. 创建访问入口，使用自动域名或自定义域名。
9. 打开访问地址验证服务。

## 二、安装应用市场模板

1. 进入应用市场。
2. 选择模板。
3. 确认镜像、官网和官方仓库。
4. 选择项目空间和运行集群。
5. 填写模板参数。
6. 保持“安装后立即部署”开启，或稍后手动部署。

## 三、查看构建与部署状态

- 构建页查看 BuildRun、BuildJob、阶段进度和日志。
- 部署页查看 Release、运行态日志、终端和回滚入口。
- 集群页查看 Kubernetes 资源、YAML 和事件。
- 账单页查看余额、用量、账本和应用花费。

## 四、常见问题

| 问题 | 可能原因 | 解决方案 |
| --- | --- | --- |
| 构建拉取基础镜像失败 | DNS 或 egress 策略阻断 | 检查 NetworkPolicy、DNS 53、镜像站白名单 |
| OIDC 登录失败 | 回调地址或准入策略错误 | 检查 Provider redirect URI、邮箱和组配置 |
| 部署一直 pending | 集群资源不足或镜像无法拉取 | 查看 Release 日志、Pod 事件和镜像凭据 |
| HTTPS 证书失败 | DNS 未解析或 issuer 不可用 | 检查域名、Ingress、cert-manager issuer |
| Secret 更新无效 | 编辑时留空表示不修改 | 输入新值后保存 |
| 账单没有生成 | 资源未运行或计费 runner 未执行 | 检查 worker、usage record 和费率规则 |
