# 初始化控制台

平台启动后，先完成最小可用配置。目标不是一次配完所有集成，而是先让控制台可登录、可创建项目空间、可继续配置运行环境。

## 登录或初始化

Compose 默认以开发模式启动 API。打开登录页后，按页面提示进入控制台。

如果你改成生产模式运行，第一次进入时访问：

```text
http://localhost:8088/bootstrap
```

这里用于创建首个管理员账号。

## 创建第一个项目空间

项目空间是团队、应用和环境的集合。你可以把它理解成一个产品或一个小团队的工作区。

建议第一次这样填：

| 字段 | 建议 |
| --- | --- |
| 名称 | 用产品名或团队名 |
| 标识 | 使用英文、小写和短横线 |
| 成员 | 先保留自己，确认流程后再邀请别人 |

项目空间列表默认展示“与我相关”的空间；平台管理员可以切换到“全部项目空间”，用于排查或全局维护。

## 创建第一个应用

应用代表一个可部署服务。第一次体验时，先创建一个空应用即可：

- 填应用名称。
- 填短一点的英文标识。
- 先不处理高级运行参数。

服务端口、镜像、Dockerfile、环境变量和数据卷都在“部署配置”里维护，不在应用基础信息里维护。

## 下一步

继续进入 [连接集群和镜像站](/guide/workspace)。

如果你已经有镜像，优先走“已有镜像部署”。这条路径最短，适合先确认平台、集群和访问入口是否正常。

如果你想从仓库自动构建，再去配置 Git Provider、镜像站和构建参数。
