DevOps 应用交付平台页面原型

这不是最终视觉稿，而是面向开发的页面说明。每个章节都带一个“截图式线框”，用于确认页面上需要有哪些区域、表单和操作。

MVP 平台构建 Job 多集群默认单集群 自定义域名 资源配额预留

03 创建应用向导

第一步选择应用来源：代码仓库或已有镜像。镜像应用不需要仓库和构建页面。

创建应用截图Step 1 / 4
来源代码仓库 / 已有镜像
基础信息名称、slug、项目
构建或镜像Dockerfile 或 image:tag
服务端口、健康检查
网关默认域名或自定义域名
代码仓库应用webhook -> BuildRun -> Builder -> image
已有镜像应用选择镜像站或填写完整镜像地址，校验失败也允许保存。
目标镜像模板变量应用配置页宽屏在表单右侧展示变量、含义和示例；窄屏自动落到表单下方。
Webhook 配置在应用配置表单内展示状态、回调地址、最近事件，并提供重新配置入口。

04 仓库绑定

跳转授权代码账户，支持多 Git 账户。绑定保存时默认由平台自动创建 webhook，失败不阻塞保存，并用 webhook secret 校验事件。

仓库绑定截图Gitea / GitHub

Webhook默认自动配置，失败可手动重试

05 应用构建 tab

构建入口位于应用详情页。主路径是平台 Builder Agent 领取 BuildJob，启动一次性 executor 容器，并用 BuildKit rootless 构建镜像。

构建页面截图BuildRun #1042
触发器push、tag、manual、api
构建资源CPU 1 核，Memory 2Gi，Timeout 20m，Retry 0
镜像推送目标应用默认 -> 项目默认 -> 全局默认
Access TokenAPI 触发构建，非 JWT，只保存 hash。

07 应用部署 tab

部署入口位于应用详情页。MVP 默认单集群体验，但模型支持多集群。Project 默认一个 namespace，环境通过资源名和 labels 区分。

部署页面截图dev / prod

08 应用访问 tab

访问入口位于应用详情页。默认域名扁平化，自定义域名通过 CNAME 验证。HTTPS 使用 HTTP Challenge，用户也可以选择 HTTP-only。

网关域名截图Ingress / Traefik
默认域名demo-api-liteyuki.dev.example.com
自定义域名api.example.org -> CNAME gateway.example.com
证书HTTP Challenge / pending_dns / active
访问模式HTTPS 或 HTTP-only

10 Access Token

用于外部系统调用平台 API 触发构建或部署。Token 不是 JWT，只保存 hash，只展示一次。
